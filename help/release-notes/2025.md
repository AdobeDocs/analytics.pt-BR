# Notas de versão de 2025 do Adobe Analytics

Saiba mais sobre as atualizações de versão mais recentes para [produtos Adobe Experience Cloud](https://business.adobe.com/br/products/adobe-experience-cloud-products.html). Obtenha a documentação de autoajuda, os tutoriais e os cursos mais recentes da Experience League.

## Julho de 2025 {#jul25}

| Recurso | Descrição | [Início da implantação](releases.md) | [Disponibilidade geral](releases.md) |
| ----------- | ---------- | ------- | ---- |
| **Campos TNT de Livestream com algoritmos** | O Livestream está passando por uma atualização para garantir que a tecnologia continue sendo moderna e estável. | **Navegação atualizada na interface de Atributos do cliente** | Agora é possível acessar a interface de Atributos do cliente diretamente a partir do seletor de aplicativos na Adobe Experience Cloud. Selecione **[!UICONTROL Atributos do cliente]** no menu suspenso (em vez de acessar a seção [!UICONTROL Pessoas] > [!UICONTROL Atributos do cliente]) e atualize seus marcadores.  A atualização inclui algumas melhorias na interface. | 1 de julho de 2025 | 31 de julho de 2025 |

| **Atualização do campo de dados de contexto do Analytics`a.locale`** | Esta atualização altera a forma como o campo de dados de contexto `a.locale` do Analytics é definido ao coletar dados via Experience Edge. Quando os dados são enviados para o Adobe Analytics usando a Experience Edge, os campos do Analytics são preenchidos com base em um mapeamento de campos XDM. O mapeamento para `c.a.locale` faz referência a um campo XDM não padrão, `xdm.environment.language`. Este campo será atualizado para fazer referência ao campo correto, `xdm.environment._dc.language`.<p>O mapeamento continuará fazendo referência a `xdm.environment.language` para oferecer compatibilidade com versões anteriores. Para fins de continuidade, se ambos os campos forem definidos, `xdm.environment.language` terá prioridade. Você pode exibir a lista completa de mapeamentos do XDM para campos padrão do Analytics [aqui](https://experienceleague.adobe.com/pt-br/docs/analytics/implementation/aep-edge/xdm-var-mapping). | | 5 de março de 2025 |
| **Guia de atualização do Customer Journey Analytics** | Permite gerar um guia passo a passo para atualizar do Adobe Analytics para o Customer Journey Analytics. | **Dimensões exclusivas do Data Warehouse** | A partir de maio de 2025, a Adobe começará a definir certas dimensões (variáveis personalizadas, como eVars e props) como “Exclusiva do Data Warehouse”. Isso se aplica às dimensões que apresentam as seguintes características:<ul><li> Durante um período de vários meses, a variável atinge o limite de tráfego baixo nos primeiros dias do mês.</li><li>> 90% dos valores transmitidos são vistos apenas uma vez durante o mês.</li></ul>Exemplos incluem dimensões que contêm carimbos de data e hora, UUIDs ou outros dados de cardinalidade extremamente alta e nos quais novos valores exclusivos são transmitidos para quase todas as ocorrências (ou visitas ou visitantes) ao longo do mês. Essas dimensões geralmente excedem os limites de tráfego baixo muito rapidamente e apenas uma parte muito pequena dos valores é relatável no Analysis Workspace. Isso torna os relatórios que utilizam essas dimensões confusos, pois não mostram informações úteis ou precisas. Essas dimensões não seguem a finalidade ou se beneficiam da funcionalidade de tráfego baixo, que tem como objetivo fornecer uma maneira de criar relatórios sobre valores que se tornam “populares” depois que a dimensão excede os limites de tráfego baixo durante o mês. [Saiba mais](https://experienceleague.adobe.com/pt-br/docs/analytics/technotes/low-traffic)<p>Dimensões marcadas como “Exclusiva do Data Warehouse” não estarão disponíveis para relatórios no Analysis Workspace. No entanto, elas ainda estarão disponíveis para relatórios por meio do Data Warehouse, pois os limites de tráfego baixo não se aplicam a elas.<p>Isso não significa que todas as dimensões que atingiram os limites de tráfego baixo sejam candidatas a serem marcadas como “Exclusiva do Data Warehouse”. A maioria das dimensões que atingem os limites de tráfego baixo atende à intenção da funcionalidade de tráfego baixo:<ul><li>A maioria dos valores transmitidos não é exclusiva.</li><li>Os valores comuns continuam a aparecer depois de atingir os limites de tráfego baixo durante o mês.</li><li>Novos valores “populares” ainda podem ocorrer.</li></ul>Somente as dimensões em que quase todos os valores transmitidos são novos e exclusivos serão marcadas como “Exclusiva do Data Warehouse”. Aumentar os limites de tráfego baixo não é uma solução, dada a singularidade dos dados coletados nessas situações. | | Maio de 2025 |
